# 研究提案：MT-MG 联合表示与意图驱动稀疏化

## 问题锚点

- **核心问题**: 当前 Video-LLM 存在严重的"语言先验幻觉"问题，缺乏细粒度时序推理能力
- **必须解决的瓶颈**: 训练数据中缺乏结构化的中间表示，导致模型过度依赖语言先验而非视觉证据
- **非目标**: 不是设计新的 Video-LLM 架构，而是通过数据增强改进现有模型
- **约束条件**:
  - 计算资源：4-8 GPU 天
  - 数据集：Ego4D、COIN、HowTo100M
  - 目标会议：NeurIPS/ICML/ICLR
- **成功条件**: 幻觉率绝对下降 >10%，同时保持或提升宏观规划和细粒度对齐性能

## 技术缺口

现有方法在程序性视频理解中存在以下问题：
1. **宏观-微观脱节**: 层次化任务结构（Ego4D Goal-Step）和场景图（EASG）被孤立使用
2. **计算复杂度**: 完整图解析是 O(N²)，对长视频不切实际
3. **幻觉问题**: 语言先验幻觉由于结构化监督不足
4. **反事实稀缺**: 缺乏系统化的硬负样本生成方法

## 方法论题

- **一句话论题**: 通过联合宏观任务树和微观场景图，并使用意图驱动的稀疏化策略，显著降低 Video-LLM 的语言先验幻觉，同时保持计算效率。

- **为什么这是最小充分干预**: 直接解决数据瓶颈，无需修改模型架构，可应用于任何 Video-LLM

- **为什么在基础模型时代及时**: 利用大语言模型生成宏观结构，利用预训练场景图模型生成微观图，符合当前多模态大模型趋势

## 贡献焦点

- **主导贡献**: MT-MG 联合表示与意图驱动稀疏化 - 首个将宏观任务结构与微观场景图联合的方法，引入意图驱动稀疏化作为归纳偏置

- **可选支持贡献**: 拓扑反事实扰动 - 在图结构层面系统化生成硬负样本

- **明确非贡献**: 不是设计新的 Video-LLM 架构，不是提出新的场景图解析算法

## 提议方法

### 复杂度预算

- **冻结/复用骨干**: Video-LLM 架构（如 VideoLLaMA、InternVideo2）
- **新训练组件**: MT-MG 联合表示编码器、稀疏化策略学习器
- **故意不使用的诱惑**: 端到端图神经网络（计算复杂度过高）、纯文本重写（缺乏结构化表示）

### 系统概览

```
输入视频
    ↓
[宏观树生成] → Macro-Tree (目标-步骤-子步骤)
    ↓
[微观图生成] → Micro-Graph (时空场景图)
    ↓
[意图驱动稀疏化] → 稀疏化 Micro-Graph
    ↓
[联合表示编码] → MT-MG 联合表示
    ↓
[QA 生成] → 结构化 QA 对
    ↓
[反事实扰动] → 硬负样本
    ↓
输出训练数据
```

### 核心机制

#### 1. 宏观树（Macro-Tree）生成

- **输入/输出**: 视频 + ASR/密集字幕 → 目标-步骤-子步骤层次结构
- **架构/策略**: 使用大语言模型（如 GPT-4）自顶向下解析，结合 ASR 和密集字幕
- **训练信号/损失**: 无监督，利用现有标注（Ego4D Goal-Step）作为弱监督
- **为什么这是主要新颖性**: 首次将宏观任务结构作为微观图解析的约束

#### 2. 微观图（Micro-Graph）生成

- **输入/输出**: 视频片段 → 时空场景图（实体-关系-状态）
- **架构/策略**: 使用预训练 VidSGG 模型和跟踪器自底向上抽取
- **训练信号/损失**: 无监督，利用预训练模型
- **为什么这是主要新颖性**: 首次将微观场景图作为宏观任务结构的验证

#### 3. 意图驱动稀疏化

- **输入/输出**: 完整 Micro-Graph + Macro-Tree 意图 → 稀疏化 Micro-Graph
- **架构/策略**: 基于宏观步骤意图过滤微观图中低相关性的冗余边，使用置信度阈值
- **训练信号/损失**: 监督学习，使用人工标注的相关性标签
- **为什么这是主要新颖性**: 首次将意图驱动稀疏化作为归纳偏置，解决计算复杂度问题

#### 4. 联合表示编码

- **输入/输出**: Macro-Tree + 稀疏化 Micro-Graph → MT-MG 联合表示
- **架构/策略**: 图神经网络编码微观图，树神经网络编码宏观树，注意力机制联合两者
- **训练信号/损失**: 监督学习，使用 QA 对作为标签
- **为什么这是主要新颖性**: 首次联合编码宏观任务结构和微观场景图

#### 5. 拓扑反事实扰动

- **输入/输出**: MT-MG 联合表示 → 扰动后的联合表示
- **架构/策略**: 在图结构层面定义合法的扰动算子（实体替换、状态阻断）
- **训练信号/损失**: 无监督，生成硬负样本
- **为什么这是主要新颖性**: 首次在图结构层面系统化生成硬负样本

### 可选支持组件

仅在需要时包含：

- **自适应阈值**: 基于置信度动态调整稀疏化阈值
- **多模态验证**: 微观证据验证宏观意图

### 现代原语使用

- **使用的 LLM/VLM/Diffusion/RL 原语**: 大语言模型（GPT-4）用于宏观树生成
- **在管道中的确切角色**: 作为宏观结构生成器，提供任务与步骤边界
- **为什么比旧式替代方案更自然**: 比手工标注更高效，比规则方法更灵活

### 集成到基础生成器/下游管道

- **新方法附加位置**: 在数据预处理阶段，生成结构化训练数据
- **冻结/可训练部分**: Video-LLM 架构冻结，MT-MG 联合表示编码器可训练
- **推理顺序**: 先生成 MT-MG 表示，再训练 Video-LLM

### 训练计划

- **分阶段或联合训练**: 分阶段训练（先训练宏观树和微观图生成器，再训练联合表示编码器）
- **损失**: QA 生成损失、稀疏化损失、反事实损失
- **数据构建**: 使用 Ego4D、COIN、HowTo100M 数据集
- **伪标签**: 使用预训练模型生成伪标签
- **课程**: 从简单任务开始，逐步增加复杂度
- **调度**: 学习率衰减、早停策略

### 失败模式和诊断

- **失败模式**: 确认偏差 - 意图驱动稀疏化过滤掉重要事件
- **如何检测**: 分析被过滤的事件，评估其对下游任务的影响
- **回退或缓解**: 设计自适应阈值和多模态验证机制

- **失败模式**: 计算复杂度过高
- **如何检测**: 监控训练时间和内存使用
- **回退或缓解**: 增加稀疏化强度，使用更高效的图神经网络

### 新颖性和优雅性论证

- **最近工作**: Ego4D Goal-Step（宏观）、EASG（微观）分别存在，但未联合
- **确切差异**: 首次联合宏观任务结构和微观场景图，引入意图驱动稀疏化
- **为什么这是聚焦的机制级贡献而非模块堆叠**: 每个组件都有明确的作用，相互补充，共同解决幻觉问题

## 声明驱动验证草图

### 声明 1: MT-MG 联合表示降低 Video-LLM 幻觉

- **最小实验**: 在 Ego4D 数据集上，比较 MT-MG 联合表示与纯文本重写、仅树、仅图基线的幻觉率
- **基线/消融**: 纯文本重写、仅树、仅图、全量图（无稀疏化）
- **指标**: 幻觉率（事实一致性）、时序推理准确性
- **预期证据**: MT-MG 联合表示幻觉率绝对下降 >10%

### 声明 2: 意图驱动稀疏化提高计算效率

- **最小实验**: 比较意图驱动稀疏化与全量图解析的计算时间和内存使用
- **基线/消融**: 全量图解析、随机稀疏化
- **指标**: 计算时间、内存使用、下游任务性能
- **预期证据**: 意图驱动稀疏化计算时间减少 >50%，性能损失 <5%

## 实验移交输入

- **必须证明的声明**:
  1. MT-MG 联合表示降低 Video-LLM 幻觉
  2. 意图驱动稀疏化提高计算效率

- **必须运行的消融**:
  1. 宏观-微观联合 vs 单独使用
  2. 意图驱动稀疏化 vs 全量图解析
  3. 拓扑反事实 vs 随机文本修改

- **关键数据集/指标**:
  - 数据集：Ego4D、COIN、HowTo100M
  - 指标：幻觉率、时序推理准确性、宏观规划能力、细粒度对齐

- **最高风险假设**:
  1. 确认偏差风险：意图驱动稀疏化可能过滤掉重要事件
  2. 计算复杂度：即使有稀疏化，图解析仍可能计算密集

## 计算和时间线估计

- **估计 GPU 小时数**: 4-8 GPU 天（32-64 GPU 小时）
- **数据/标注成本**: 低，使用现有数据集和预训练模型
- **时间线**: 3-4 个月（1 个月实现，1-2 个月实验，1 个月分析）
