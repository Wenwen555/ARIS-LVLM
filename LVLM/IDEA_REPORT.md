- 但现有工作主要专注于**模型架构**，缺乏**数据中心化**的自动化数据构建管道
- **意图驱动稀疏化** 是我们的核心差异化，解决长视频 O(N²) 难题

该方法通过联合宏观任务树和微观场景图，并使用意图驱动的稀疏化策略，显著降低 Video-LLM 的语言先验幻觉，同时保持计算效率。审查评分 6.5/10，需要进一步细化机制和设计实验。

## 文献景观（2024-2026 更新）

当前程序性视频理解领域呈现出三个关键趋势：

1. **从扁平描述到联合表示**: 从 YouCook2 的时序步骤边界发展到 VideoLoom 的时空联合理解
2. **从模型中心到数据中心**: 大部分工作专注于模型架构，但数据构建管道是关键瓶颈
3. **计算效率成为关键**: 联合表示的 O(N²) 复杂度对长视频不切实际

**最新工作**:
- **VideoLoom (2025)**: 时空联合理解，但专注于模型架构，计算复杂度高
- **EASG (2024)**: 视角动作场景图，微观图表示
- **State-Change Counterfactuals (ICCV 2025)**: 状态变化反事实

**我们的定位**:
- 不同于 VideoLoom 的模型架构焦点，我们专注于**数据中心化的自动化数据构建管道**
- **意图驱动稀疏化** 解决长视频可扩展性问题

## 排名想法

### 🏆 想法 1: MT-MG 联合表示与意图驱动稀疏化 — 推荐

- **假设**: 通过联合宏观任务树和微观场景图，并使用意图驱动的稀疏化策略，可以显著降低 Video-LLM 的语言先验幻觉，同时保持计算效率。

- **最小实验**: 在 Ego4D 或 COIN 数据集上构建 MT-MG 表示，训练 Video-LLM，与纯文本基线、仅树基线、仅图基线、VideoLoom-style 联合表示基线比较。评估宏观规划能力（任务预测）、细粒度对齐（时序定位）、幻觉率（事实一致性）和计算效率。

- **预期结果**: MT-MG + 稀疏化在所有指标上应优于单独使用树或图，幻觉率显著降低（绝对下降 >10%），计算效率高于联合表示基线（O(N log N) vs O(N²)）。

- **新颖性**: 6.5/10 — VideoLoom (2025) 是最接近工作，但我们的贡献在于数据中心定位和意图驱动稀疏化；Ego4D Goal-Step（宏观）和 EASG（微观）分别存在，但未联合；State-Change Counterfactuals（ICCV 2025）涵盖状态变化但非拓扑扰动。

- **差异化**:
  - **vs VideoLoom**: 数据中心 vs 模型架构；意图稀疏化 O(N log N) vs O(N²)；显式宏观-微观耦合
  - **vs EASG**: 增加宏观任务结构
  - **vs State-Change Counterfactuals**: 拓扑层面 vs 状态层面

- **可行性**:
  - 数据：Ego4D、COIN、HowTo100M 可用
  - 计算：需要 4-8 GPU 天用于完整实验
  - 实现：中等复杂度，需要图解析和层次结构建模

- **风险**: MEDIUM-HIGH — VideoLoom 是强竞争者；确认偏差风险；稀疏化策略可能过滤重要事件

- **贡献类型**: 方法 + 实证发现 + 数据中心管道

### 想法 2: 拓扑反事实扰动用于硬负样本生成 — 备份

- **假设**: 通过在图结构层面定义合法的扰动算子（实体替换、状态阻断），生成的硬负样本可以强制 Video-LLM 依赖视觉证据而非语言共现性，从而显著降低幻觉率。

- **最小实验**: 在程序性视频数据集上，使用拓扑反事实生成 QA 对，训练 Video-LLM，与随机文本修改基线、状态变化反事实基线比较。评估幻觉率、时序推理准确性和对抗鲁棒性。

- **预期结果**: 拓扑反事实训练的模型幻觉率应显著低于基线（绝对下降 >15%），在对抗性问题上表现更好。

- **新颖性**: 7/10 — State-Change Counterfactuals（ICCV 2025）涵盖状态变化但非拓扑扰动；CaptainCook4D 包含错误执行但未系统化生成反事实。

- **可行性**:
  - 数据：需要程序性视频 + 图解析
  - 计算：需要 2-4 GPU 天
  - 实现：中等复杂度，需要图操作和扰动算子

- **风险**: MEDIUM — 拓扑扰动可能生成不合理的场景，影响训练质量

- **贡献类型**: 方法 + 实证发现

- **试点结果**: SKIPPED: 需要 GPU

- **为什么我们应该做这个**: 这是第一个在图结构层面系统化生成硬负样本的方法，超越了传统的文本域随机修改和状态变化反事实。

### 想法 3: 自适应稀疏化策略与多模态验证 — 改进

- **假设**: 通过动态调整稀疏化阈值并交叉验证宏观意图与微观证据，可以在保持计算效率的同时减少确认偏差，提高 MT-MG 表示的鲁棒性。

- **最小实验**: 在 MT-MG 框架中实现自适应稀疏化（基于置信度动态调整阈值）和多模态验证（微观证据验证宏观意图），与固定阈值基线比较。评估稀疏化效率、保留事件准确性和下游任务性能。

- **预期结果**: 自适应稀疏化应在保持计算效率的同时，保留更多真实事件（召回率提升 >10%），下游任务性能优于固定阈值。

- **新颖性**: 6/10 — Video-Mined Task Graphs 使用概率图但无稀疏化；现有稀疏化方法多为固定阈值。

- **可行性**:
  - 数据：需要程序性视频 + MT-MG 表示
  - 计算：需要 2-3 GPU 天
  - 实现：中等复杂度，需要置信度估计和动态阈值调整

- **风险**: LOW — 方法相对保守，即使失败也可回退到固定阈值

- **贡献类型**: 方法改进

- **试点结果**: SKIPPED: 需要 GPU

- **审稿人可能的反对意见**: "自适应策略可能增加计算开销，抵消稀疏化的效率优势"

- **为什么我们应该做这个**: 这解决了 MT-MG 中确认偏差风险的核心问题，提高了方法的鲁棒性。

## 细化提案

### 方法论题

通过联合宏观任务树和微观场景图，并使用意图驱动的稀疏化策略，显著降低 Video-LLM 的语言先验幻觉，同时保持计算效率。与 VideoLoom 等模型架构工作形成互补，专注于数据中心化的自动化数据构建管道。

### 主导贡献

MT-MG 联合表示与意图驱动稀疏化 - 首个将宏观任务结构与微观场景图**显式耦合**（意图驱动）的方法，引入意图驱动稀疏化作为归纳偏置，解决长视频 O(N²) 可扩展性难题。

### 核心机制

1. **宏观树（Macro-Tree）生成**: 使用大语言模型自顶向下解析，生成目标-步骤-子步骤层次结构
2. **微观图（Micro-Graph）生成**: 使用预训练 VidSGG 模型和跟踪器自底向上抽取时空场景图
3. **意图驱动稀疏化**: 基于宏观步骤意图过滤微观图中低相关性的冗余边（核心创新）
4. **联合表示编码**: 图神经网络编码微观图，树神经网络编码宏观树，注意力机制联合两者
5. **拓扑反事实扰动**: 在图结构层面定义合法的扰动算子，生成硬负样本

### 与 VideoLoom 的差异化

| 维度 | VideoLoom | MT-MG |
|------|-----------|-------|
| **焦点** | 模型架构 | 数据构建管道 |
| **稀疏化** | 无（O(N²) 复杂度） | 意图驱动（O(N log N)） |
| **宏观结构** | 隐式 | 显式 goal-step-substep |
| **耦合机制** | 隐式联合 | 意图驱动显式耦合 |
| **反事实** | 无 | 拓扑反事实 |

### 必须证明的声明

1. MT-MG 联合表示降低 Video-LLM 幻觉
2. 意图驱动稀疏化提高计算效率（O(N log N) vs O(N²)）
3. 数据中心管道在相同模型架构下优于纯文本数据

## 关键技术组件

### 宏观树（Macro-Tree）
- 层次结构：目标-步骤-子步骤
- 生成方式：通用大语言模型 + ASR/密集字幕自顶向下解析
- 作用：提供任务与步骤边界，约束微观图解析

### 微观图（Micro-Graph）
- 结构：局部时空场景图
- 生成方式：预训练 VidSGG 模型和跟踪器自底向上抽取
- 作用：捕获细粒度实体交互和状态变化

### 意图驱动稀疏化 ⭐ 核心创新
- 策略：利用宏观步骤意图过滤微观图中低相关性的冗余边
- 权衡：大幅压缩 O(N²) 图节点规模到 O(N log N)
- 风险缓解：设置置信度阈值和保留高频背景节点
- 与 VideoLoom 区分：VideoLoom 无显式稀疏化策略

### 拓扑反事实扰动
- 算子：实体替换、状态阻断等合法图结构修改
- 目标：生成硬负样本，强制模型依赖视觉证据
- 区别：区别于纯文本域的随机修改和状态变化反事实，在图结构层面操作
