## Main Remaining Critiques

### 1. Self-produced evidence is a weak integration point (CRITICAL)

**Issue**: Right now `VideoLLaMA-2 → candidate answers + evidence spans`, then `SSV` verifies those spans. This creates a **self-justification loop**: the same model can hallucinate both the answer and the evidence trace. If that happens, SSV may learn consistency with the model's own narration rather than true visual support.

**Fix**: Define evidence spans as **retrieved temporal segments from the video encoder**, not just self-reported textual rationales from VideoLLaMA-2.

### 2. Positive labels are too optimistic (CRITICAL)

**Issue**: "Real QA pair = support sufficient" is not a safe assumption. In procedural-video datasets, some answers are correct but only weakly grounded, ambiguous, or recoverable from script priors. If those are treated as clean positives, SSV will learn a noisy notion of sufficiency.

**Fix**: Use the 200-300 human-verified subset to supervise sufficiency and attribution, and treat the rest as weak supervision.

### 3. MECG validation underspecified (CRITICAL)

**Issue**: "Verify answer flip" is still too vague. If the flip is checked by heuristics or by the same base model, the hard negatives may be noisy or circular. For abstention training, you do not actually need a clean alternative answer; you only need a trustworthy "support destroyed / insufficient" label.

**Fix**: Remove one support-critical segment or tuple, require that the edited trace no longer contains the minimal evidence set, and label it as insufficient. Do not hinge the paper on proving a new answer flip.

### 4. Pseudo-novelty risk (IMPORTANT)

**Issue**: Verifier + abstain can read as standard selective QA. Reviewers may see this as a standard answer verifier or reranker with abstention.

**Fix**: The novelty must be explicit: **insufficiency-specific training using minimal support corruption in procedural video QA**.

### 5. Text-only serialized trace may be too lossy (IMPORTANT)

**Issue**: If purely text-serialized, the model may collapse into textual NLI over noisy tuples.

**Fix**: Include segment IDs and temporal order markers explicitly. Consider cross-attention over frozen visual segment features.
